Retrieval-Augmented Generation (RAG) is a technique that combines information retrieval with large language models (LLMs) to answer questions using both external documents and generative reasoning.

Key RAG interview topics:
- How RAG retrieves relevant documents from a vector database
- The role of embeddings in semantic search
- How the LLM integrates retrieved context into its answer
- Challenges: context window limits, hallucination, citation tracking

Agentic RAG extends RAG by introducing agents that can use tools, reason step-by-step, and perform multi-hop retrieval. This enables more complex workflows and dynamic information gathering.

Agentic RAG interview Q&A:
Q: What is the main advantage of agentic RAG over traditional RAG?
A: Agentic RAG enables multi-step reasoning and tool use, allowing the system to answer more complex queries and interact with multiple data sources.

Q: How do you prevent hallucination in RAG systems?
A: By grounding answers in retrieved documents, using citation tracking, and limiting the LLM to only use provided context.

Q: What are common use cases for RAG in industry?
A: Enterprise search, customer support, knowledge management, and technical Q&A systems.
